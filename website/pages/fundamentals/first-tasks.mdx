## 1.3: 你的第一个 PM 任务

 - **完成时间：** 30 分钟
 - **先修要求：** 完成模块 1.1 和 1.2


> **在 Claude Code 中启动这个模块：** 运行 `/start-1-3` 来开始互动体验。

## 概述

模块 1.3 教你使用 Claude Code 进行真实的 PM 工作：处理会议记录、综合研究、分析视觉内容，并创建可重用模板。这是理论变为实践的地方。

**关键要点：** Claude Code 是一个文件感知、视觉能力的助手，可以每周为你节省 2-3 小时的常规 PM 文档任务。


## 核心能力

本模块涵盖五项基本技能：

| 能力 | 你将学到什么 |
|------------|-------------------|
| **文件读取/写入** | 使用 `@` 引用文件、读取多个文档、创建/编辑文件 |
| **内容分析** | 将笔记转化为行动项、综合研究、提取模式 |
| **图像粘贴** | 使用 Ctrl+V 直接将截图粘贴到终端（所有平台） |
| **视觉分析** | 获得模型反馈、分析竞争对手、审核数据可视化 |
| **输出样式** | 为重复的文档类型创建可重用的格式模板 |


## @ 符号 - 文件引用系统

`@` 符号告诉 Claude 要读取、分析或引用哪些文件。

### 基本语法

```
读取 @meeting-notes.txt 并总结行动项
```

```
比较 @competitor-a.md 和 @competitor-b.md
```

```
分析 @research/interviews/user-01.txt
```

### 支持的文件类型

| 格式 | 扩展名 | 常见用途 |
|--------|-----------|-------------|
| Markdown | `.md` | PRD、文档、笔记 |
| Text | `.txt` | 会议记录、原始数据 |
| CSV | `.csv` | 调查结果、指标 |
| JSON | `.json` | API 响应、结构化数据 |
| PDF | `.pdf` | 研究论文、报告 |
| Images | `.png`, `.jpg`, `.gif` | 截图、模型图 |

### 处理多个文件

**批量处理文件：**
```
读取 /meetings/ 中的所有文件并提取行动项
```

**交叉引用文档：**
```
读取 @prd.md 和 @user-research.md。
识别 PRD 中未解决用户痛点的空白处。
```

**从多个来源综合：**
```
读取 /interviews/ 中的所有文件并创建 @research-synthesis.md，包含：
- 前5个痛点
- 常见功能请求
- 用户画像
```


## 粘贴图像 - 关键键盘快捷键

### 通用命令：Ctrl+V

**重要：在所有平台上都使用 Ctrl+V，包括 Mac**

| 平台 | 正确 | 错误 |
|----------|---------|-------|
| Mac | **Ctrl+V** | Command+V |
| Windows | **Ctrl+V** | - |
| Linux | **Ctrl+V** | - |

**为什么 Mac 不同？** 在终端环境中，`Ctrl+V` 在所有操作系统中都是通用的。`Command+V` 是 Mac 的 GUI 粘贴命令，在终端环境中不起作用。

### 如何粘贴图像

1. **将图像复制到剪贴板**
   - 截图：`Cmd+Shift+4` (Mac) 或 `Win+Shift+S` (Windows)
   - 从浏览器/Figma 复制：`Cmd+C` (Mac) 或 `Ctrl+C` (Windows)

2. **粘贴到 Claude Code**
   - 在终端中按 **Ctrl+V**
   - 图像出现在对话中

3. **请求分析**
   ```
   分析这个模型图并提供：
   - PM 视角的 UX 反馈
   - 技术挑战
   - 可访问性考虑
   - 缺失的用户流程元素
   ```

### 你可以粘贴什么

| 类别 | 示例 |
|----------|----------|
| **设计与产品** | Figma 模型图、线框图、用户旅程 |
| **数据与分析** | 仪表板截图、图表、指标 |
| **竞争研究** | 竞争对手截图、定价页面 |
| **协作** | 白板照片、会议图表 |
| **技术** | 错误消息、架构图 |


## 内容转换

PM 的核心技能是向不同受众传达相同的信息。

### 沟通矩阵

| 受众 | 格式 | 语气 | 重点 |
|----------|--------|------|-------|
| 工程 | Slack | 随意、技术性 | 实现细节 |
| 利益相关者 | 邮件 | 专业、战略性 | 业务成果 |
| 客户 | 发布说明 | 友好、以利益为中心 | 交付的价值 |
| 文档 | Notion 文档 | 全面、结构化 | 完整参考 |
| 高管 | 简报 | 简洁、以指标为驱动 | 影响和下一步 |

### 示例：多格式沟通

```
读取 @sprint-planning-notes.txt 并创建：
1. @slack-update.md - 随意的团队公告
2. @stakeholder-email.md - 专业的高管总结
3. @notion-doc.md - 完整的参考文档
```


## 输出样式 - 可重用模板

输出样式是消除重复格式化解释的格式模板。

### 创建输出样式

**方法1：定义格式**
```
创建高管简报输出样式，包含：
- 最多3个段落
- 段落1：发生了什么（结果、指标）
- 段落2：为什么重要（业务影响）
- 段落3：下一步（前瞻性）
- 对指标使用项目符号
- 无行话，主动语态
```

**方法2：显示示例**
```
创建格式如下的用户故事输出样式：

**作为** [画像]
**我想要** [能力]
**以便** [利益]

**验收标准：**
- 给定 [上下文]
  当 [行动]
  然后 [结果]
```

### 使用输出样式

```
将 @meeting-notes.txt 转换为高管简报样式
```

```
将 @sprint-work.md 格式化为每周更新样式
```

### 基本 PM 输出样式

使用这些格式构建你的库：

1. **高管简报** - 3段式战略总结
2. **用户故事** - 作为/我想要/以便 并带有验收标准
3. **Linear/Jira 问题** - 标题、描述、验收标准、优先级、估算
4. **每周更新** - 已发布/进行中/受阻/下周计划
5. **发布说明** - 面向客户、以利益为中心
6. **PRD 部分** - 问题/解决方案/指标/故事
7. **Slack 公告** - 随意、友好表情符号
8. **利益相关者邮件** - 专业、内容丰富


## 真实 PM 工作流程

### 会议处理（每周）

**不使用 Claude Code：** 35分钟
- 审查笔记：10分钟
- 提取行动项：15分钟
- 按负责人组织：5分钟
- 格式化以便分发：5分钟

**使用 Claude Code：** 2分钟
```
处理 /meetings/today/ 中的所有文件并创建 @action-items-summary.md，包含：
- 按负责人组织的行动项表
- 优先级和截止日期
- 按会议来源分组
```

**节省时间：** 33分钟/周 = 2.75小时/月

### 用户研究综合

**不使用 Claude Code：** 2小时40分钟
- 阅读8份记录：60分钟
- 突出主题：30分钟
- 提取引用：20分钟
- 组织发现：20分钟
- 撰写综合：30分钟

**使用 Claude Code：** 5分钟
```
读取 /user-interviews/ 中的所有文件并创建 @research-synthesis.md，包含：
- 前5个痛点（3+用户提及）
- 每个发现的支持引用
- 按频率排序的功能请求
- 带有真实数据的精炼画像
- 推荐的下一步
```

**节省时间：** 每个研究周期2.5小时

### 利益相关者沟通

**不使用 Claude Code：** 45分钟
- 起草 Slack 消息：10分钟
- 撰写利益相关者邮件：15分钟
- 创建 Notion 文档：15分钟
- 确保一致性：5分钟

**使用 Claude Code：** 3分钟
```
读取 @product-sync-notes.md 并创建三种格式：

1. Slack 更新 - 随意、友好表情符号、庆祝胜利
2. 利益相关者邮件 - 业务成果、指标、风险
3. Notion 文档 - 带有依赖项的完整参考
```

**节省时间：** 42分钟/冲刺 = 1.4小时/月


## 最佳实践

### 要做：

**明确使用 @ 引用**
```
好：读取 @user-research.md 并总结痛点
更好：读取 @user-research.md 并创建 @pain-points-summary.md
```

**请求结构化输出**
```
创建表：行动项 | 负责人 | 截止日期 | 优先级
```

**为分析提供上下文**
```
我们正在决定Q2使用功能A还是B。
读取 @user-research.md 并推荐哪个解决更关键的痛点。
```

**组合多个操作**
```
读取所有 /interviews/，综合发现，创建 @executive-summary.md 和 @detailed-insights.md
```

**逐步构建输出样式库**
- 在遇到重复需求时创建样式
- 不要一次性创建所有样式


### 不要做：

**忘记 @ 符号**
```
坏：总结 meeting-notes.txt
好：总结 @meeting-notes.txt
```

**在 Mac 上使用 Command+V**
- 图像始终使用 Ctrl+V（所有平台）

**期望 Claude 记住未保存的工作**
- 用文件名保存文件以供后续引用

**粘贴敏感信息**
- 先从截图中编辑机密数据

**创建过于复杂的样式**
- 保持简单和专注
- 更好：10个简单样式而不是1个复杂的


## 故障排除

**Command+V 在 Mac 上不起作用**
- 解决方案：改用 **Ctrl+V**
- 原因：终端在所有平台上都使用 Ctrl 快捷键

**Claude 找不到引用的文件**
- 检查文件名拼写（区分大小写）
- 验证文件路径：`/meetings/ 中有什么文件？`
- 询问 Claude：`@meeting-notes.txt 存在吗？`

**图像粘贴了但没有分析**
- 验证图像出现在对话中
- 尝试不同格式（.png 或 .jpg）
- 检查大小（保持在10MB以下）
- 用 Ctrl+V 重新粘贴

**输出样式格式不正确**
- 显示所需格式的具体示例
- 具体说明结构
- 迭代测试和改进
- 从简单开始，逐步增加复杂性

**文件创建了但在 Obsidian 中不可见**
- 刷新 Obsidian
- 检查不在隐藏的 .claude/ 文件夹中
- 在 Finder/Explorer 中查找
- 询问 Claude：`你把文件保存在哪里了？`
