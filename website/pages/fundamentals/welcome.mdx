## 1.1: Welcome to TaskFlow

 - **Time to Complete:** 20 minutes
 - **Prerequisites:** Claude Code installed


> **Start this module in Claude Code:** Run `/start-1-1` to kick off the interactive experience.

## ▶️ 开始使用提醒

⚠️ **如果你还没有克隆仓库：** 请先回到 [模块 0.2：启动 Claude Code 并克隆课程](/getting-started/start-and-clone)。你需要在电脑上安装课程文件才能继续。


**在 Claude Code 中启动本课程有两种方式：**

### 选项一：在文件资源管理器中查找

导航到你的文档文件夹，找到 `claude-code-pm-course` 文件夹。然后右键点击并选择"在文件夹中新建终端"

看起来像这样：

![从文件资源管理器打开文件夹](/images/folder-location.png)

然后输入 `claude` 并按回车

### 选项二：从终端打开

在终端中输入这些命令：
```bash
cd ~/Documents/claude-code-pm-course
claude
```

这会导航到你的课程文件夹并直接在其中打开 Claude Code。

**无论哪种方式，你都会到达同一个地方。然后，要启动这个模块：**

> **在 Claude Code 中启动这个模块：** 运行 `/start-1-1` 来开始互动体验。

这就是你启动每个模块的方式！

## 📖 概述

这个模块介绍 TaskFlow（你的虚拟公司）并解释课程结构。

**关键要点：** 这是一个动手实践课程，你将使用 Claude Code 进行真实的 PM 工作。


## 🎓 课程如何运作

### 两个部分

**1. 互动模块（你现在正在做的）**
- 使用 Claude Code 完成模块
- 每个模块以斜杠命令开始：`/start-1-1`、`/start-1-2` 等
- Claude 一步步指导你完成练习
- 你实际执行工作（不仅仅是阅读）
- 为 TaskFlow 进行真实的 PM 任务

**2. 参考指南（你现在正在阅读的）**
- 每个模块的综合独立指南
- 可以在网站上阅读，无需使用 Claude Code
- 当你需要更深入的解释时使用
- 非常适合复习和后续参考

### 课程结构

- **模块 1：基础知识** - 核心概念和工作流
- **模块 2：高级内容** - 真实的 PM 工作（PRD、数据分析、策略）

## 🏢 什么是 TaskFlow？

### 公司概述

**TaskFlow** 是为本课程创建的一个虚构的项目管理 SaaS 公司。想象一下 **Asana 遇上 Jira，为远程优先团队打造。**

**公司详情：**
- **成立时间：** 2021 年
- **阶段：** B 轮创业公司
- **融资：** 2000 万美元
- **团队规模：** 50 名员工（15 名工程，5 名产品，10 名设计，20 名 GTM）
- **指标：** 250 万美元 ARR，10,000 名活跃用户，快速增长

**你的角色：** 高级产品经理
- 你负责激活和 onboarding
- 向 CEO（Sarah Chen）汇报
- 与 CTO（Mike Rodriguez）和设计总监（Alex Kim）密切合作

**使命：** 赋能远程团队无缝协作

**产品：** 项目管理工具，结合任务管理、团队协作和异步优先功能，专为远程团队打造

**关键差异化：** 比 Asana 更简单，比 Linear 更跨职能，专为异步工作打造

### 👥 用户角色

- **Sarah - 企业管理员：** 大公司的 IT 负责人，专注于安全、合规和成本控制

- **Mike - 个人贡献工程师：** 软件工程师，专注于快速工具、GitHub 集成和最小干扰

- **Alex - 团队负责人：** 工程经理，专注于团队可见性、工作负载平衡和进度跟踪


## 📂 课程文件结构

当你在 TaskFlow 课程中工作时，你会发现以下内容：

```
claude-code-pm-course/
├── company-context/
│   ├── COMPANY.md - 公司概述和你的角色
│   ├── PRODUCT.md - TaskFlow 的功能
│   ├── PERSONAS.md - 3 个用户角色
│   └── COMPETITIVE.md - 竞争格局
├── lesson-modules/
│   ├── 1.1-welcome/ - 你现在所在的位置！
│   │   ├── CLAUDE.md (Claude 的教学脚本)
│   │   └── REFERENCE_GUIDE.md (这个文件)
│   ├── 1.2-visualizing-files/
│   ├── 1.3-reading-files/
│   └── [更多模块...]
├── your-work/
│   └── [你在练习中创建的文件]
└── CLAUDE.md - 项目记忆（在飞书文档中可见）
```

**注意：** `.claude/` 文件夹（用于命令、代理、设置）存在但被隐藏。你将在后面的模块中使用它。

## ⏱️ 时间投入

**每个模块：** 15-30 分钟
**每个级别：** 2-4 小时
**完整课程：** 20-30 小时

**但是：** 你可以按照自己的节奏进行！
- 一次完成一个模块
- 在级别之间休息
- 随时回来（进度会被保存）

### 你将看到的内容

在整个课程中，你会看到 Claude 运行类似这样的命令：

```
⏺ Read(company-context/COMPANY.md)
  ⎿  Read 45 lines

⏺ Bash(ls -la company-context/)
  ⎿  Listed 4 files
```

**不用担心理解这些！** 这些是 Claude 在后台工作。你只需要专注于 PM 工作。

### 权限提示

偶尔，你会看到询问的提示：
你想允许这个命令吗？

**只需点击：** 是，不要再询问

这让 Claude 能够顺畅工作，而不必每次都询问权限。


## 📋 最佳实践

## ❓ 常见问题

**我需要安装什么吗？**
只需要飞书文档（模块 1.2），它是免费的。

**我可以跳过模块吗？**
不推荐。每个模块都建立在前面模块的基础上。

**完整课程需要多长时间？**
总共 15-25 小时，按照你自己的节奏。

**这适合非技术 PM 吗？**
是的！不需要编码。

**我可以在实际工作中使用这些吗？**
绝对可以。完成模块 1 后，你可以将这些模式应用到你的实际 PM 工作中。
