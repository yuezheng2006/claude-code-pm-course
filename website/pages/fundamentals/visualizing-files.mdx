## 1.2: 本地文件可视化

 - **完成时间：** 15-20 分钟
 - **前提要求：** 模块 1.1 完成，基本熟悉文件系统


> **在 Claude Code 中开始此模块：** 运行 `/start-1-2` 开始互动体验。

## 📖 概述

设置本地文件可视化工具以在 Claude Code 旁边实时可视化您的项目文件，实现无缝的分屏工作流程。

**关键要点：** 本地可视化工具让您观看文件在 Claude 工作时出现和更新，无需依赖网络连接。


## 🎯 为什么可视化很重要

在终端中使用 Claude Code 感觉像是在盲目工作。您要求 Claude 创建文档，但您看不到正在发生什么。

**解决方案：** 在终端旁边打开本地文件可视化工具。现在您可以：
- 观看 Claude 创建文件时的文件出现
- 在 Claude 工作时阅读文档
- 验证一切是否正确组织
- 始终确切知道正在发生什么

这将 Claude Code 从一个神秘的黑箱转换为一个透明的工具，您可以实时观看和验证。


## 🎨 推荐的本地工具

选择适合你需求的本地文件可视化工具，以下是推荐选项：

### VS Code (推荐新手)
- 免费且功能强大
- 内置文件浏览器
- 支持丰富的文件格式
- 强大的搜索和编辑功能

### Obsidian (推荐给PM)
- 专注于笔记和知识管理
- 支持双链和知识图谱
- 非常适合产品文档管理
- 优秀的markdown支持

### Cursor
- AI驱动的代码编辑器
- 与Claude Code配合良好
- 基于VS Code构建

### 系统文件管理器
- **Mac:** Finder
- **Windows:** 文件资源管理器
- 简单直接，无需安装

选择你觉得最舒适的工具！


## 🚀 设置指南

**下载工具：**

### VS Code 设置
1. 访问 https://code.visualstudio.com/
2. 下载并安装适合您系统的版本
3. 打开 VS Code
4. 选择 `File > Open Folder...`
5. 导航到您的课程文件夹并选择

### Obsidian 设置
1. 访问 https://obsidian.md/
2. 下载并安装
3. 打开 Obsidian
4. 选择"打开文件夹作为仓库"
5. 选择您的课程文件夹

**分屏工作流程：** 将 Claude Code 终端和您的本地工具并排放置，以实时观看文件变化


## ⚠️ .claude/ 文件夹

`.claude/` 文件夹是一个特殊的隐藏文件夹（以 `.` 开头），包含：
- 自定义命令（斜杠命令）
- 自定义子代理
- 设置和配置

**在本地工具中查看 `.claude/` 文件：**

### VS Code
- 默认显示所有文件夹，包括隐藏文件夹

### Obsidian
- 设置 > 文件与链接 > 显示隐藏文件

### 系统文件管理器
- **Mac：** 在 Finder 中按 `Cmd + Shift + .` 显示隐藏文件夹
- **Windows：** 在查看选项卡中启用"隐藏的项目"

**何时需要它：** 后续模块中会使用。现在，专注于常规项目文件。


## 🎬 实时可视化演示

2. 在 Claude Code 中询问：*创建一个名为 test-visualization.md 的文件并添加一些内容*

## 💡 本地工具使用技巧

### VS Code 技巧
- **快速文件搜索** - `Cmd+P` (Mac) 或 `Ctrl+P` (Windows)
- **全局搜索** - `Cmd+Shift+F` 搜索所有文件内容
- **文件浏览器** - `Cmd+B` 切换文件浏览器
- **分屏编辑** - 查看多个文件

### Obsidian 技巧
- **快速打开** - `Cmd+O` 快速打开文件
- **全局搜索** - `Cmd+Shift+F` 搜索所有文件
- **双链** - 使用 `[[]]` 连接相关文档
- **图谱视图** - 可视化文档关系

### 通用技巧
- **文档组织** - 使用文件夹和子文件夹按项目和时间线组织
- **标签系统** - 使用 `#prd #user-research #q1-2025` 进行分类
- **搜索功能** - 快速找到需要的信息

## 🐛 故障排除

### 我看不到任何文件

**可能原因：** 打开了错误的文件夹

**修复方法：**
1. 询问 Claude：我的课程文件夹路径是什么？
2. 在本地工具中：File > Open Folder（或等效操作）
3. 再次选择文件夹，使用 Claude 给的确切路径
4. 你的文件现在应该会出现

**常见错误：** 打开了父文件夹而不是项目文件夹本身

**示例：**
- ❌ 打开了：`/Users/yourname/Documents`（层级太高）
- ✅ 应该打开：`/Users/yourname/Documents/claude-code-pm-course/course-materials`

### 文件没有实时更新

**可能原因：** 本地工具需要刷新

**修复方法：**
- **VS Code:** 文件会自动更新，通常无需手动刷新
- **Obsidian:** 文件会自动同步
- **文件管理器:** 按 `Cmd+R` (Mac) 或 `F5` (Windows) 刷新

**预防：** 大多数现代工具都会自动检测文件变化。

### .claude/ 文件夹在哪里？我找不到它！

**答案：** 它是隐藏的！这是正常且预期的。

**要查看它：**

**VS Code:**
- 默认显示所有文件夹，包括隐藏文件夹

**Obsidian:**
- 设置 > 文件与链接 > 显示隐藏文件

**Mac Finder:**
1. 导航到你的项目文件夹
2. 按：`Cmd + Shift + .`（句点）
3. `.claude/` 变得可见

**Windows 文件资源管理器:**
1. 导航到你的项目文件夹
2. 查看选项卡 → 选中隐藏的项目
3. `.claude/` 变得可见

### 哪个工具最适合我？

**如果你是编程新手：** 推荐 VS Code，界面直观，功能丰富

**如果你专注于文档管理：** 推荐 Obsidian，专为笔记和知识管理设计

**如果你想要AI辅助：** 推荐 Cursor，内置AI功能

**如果你只需要简单查看：** 使用系统文件管理器即可
