# PM 工作流示例

想象一下这个典型的 PM 工作流:

**上午:**

- 让 Claude 总结昨天的会议记录
- 在本地文件管理器（Finder/文件资源管理器）或文本编辑器（VS Code、Cursor等）中看到总结文件出现
- 打开文件阅读,如有需要进行编辑
- 如果需要分享,可以手动上传到飞书文档或直接发送给团队

**下午:**

- 让 Claude 根据用户研究起草 PRD
- 在本地编辑器中实时看到 PRD 文件被创建和更新
- 分屏查看:左侧与 Claude 聊天,右侧在编辑器中查看文件
- 请求修订:"让第 3 部分更技术化"
- 在编辑器中实时观看文件变化

**一天结束:**

- 让 Claude 处理今天的所有行动项
- 在本地文件系统中看到整理好的列表文件出现
- 确切知道明天要做什么

**重要提示:**

- Claude Code 直接在本地文件系统中创建和编辑文件
- 使用本地文件管理器或文本编辑器（VS Code、Cursor、Obsidian等）来查看文件
- 如果需要使用飞书文档,可以手动上传文件,但这不是实时同步的
- 关键是:你始终能在本地看到 Claude 正在创建和修改的文件

这就是你的新现实!你与 Claude 一起工作,看到一切,始终掌控一切。
