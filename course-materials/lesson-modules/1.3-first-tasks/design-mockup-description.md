概述:

Jordan 为 SingTech 展示了最终的暗模式模型。这是版本 3,在整合来自工程和产品团队的反馈之后。该设计涵盖了主要应用程序屏幕:项目板、任务详细信息视图、导航和设置。

颜色板:

背景颜色:
- 主背景:#1a1a1a(深灰色,不是纯黑色)
- 次要背景:#242424(稍浅用于卡/面板)
- 三级背景:#2e2e2e(用于悬停状态)

文本颜色:
- 主文本:#f0f0f0(离白色,不是纯白色 - 减少眼睛疲劳)
- 次文本:#b0b0b0(较少重要文本的灰色)
- 禁用文本:#666666(较深灰色)

重音颜色:
- 主品牌(蓝色):#4d9fff(比浅色模式 #2563eb 更浅)
- 成功(绿色):#4ade80(比浅色模式更浅)
- 警告(黄色):#fbbf24(与浅色模式相同,已经很亮)
- 错误(红色):#f87171(比浅色模式更浅)
- 链接:#60a5fa(较浅蓝色,与主品牌不同)

边框/分隔符:
- 微妙的边框:#333333(几乎看不见,刚好足以分开)
- 强边框:#444444(用于强调分离)

值得注意的设计决定:

对比度比率:
Jordan 验证了所有文本都符合 WCAG AA 标准(正常文本 4.5:1,大文本 3:1)。主文本(#1a1a1a 上的 #f0f0f0)的比率为 11.5:1。次文本(#1a1a1a 上的 #b0b0b0)的比率为 6.2:1。

不是纯黑色:
"我使用 #1a1a1a 而不是 #000000,因为纯黑色在 OLED 屏幕上,带有纯白色文本会导致眼睛疲劳。稍浅的黑色更容易眼睛。"

语法突出显示:
任务描述中的代码块使用深色语法主题(类似于 VS Code 的 Dark+ 主题)。Jordan 包含了 JavaScript、Python 和 JSON 的示例。

用户上传的图像:
任务附件中的图像具有细微的 #333333 边框,以将其与背景分开。Jordan 指出: "如果用户上传带有白色背景的屏幕截图,它们看起来会很奇怪。边框有帮助。"

图标:
所有图标都是 #f0f0f0 或 #b0b0b0,取决于重要性。Jordan 调整了几个在暗模式中看起来很微弱的图标 - 使它们稍微更大胆。

交互状态:

悬停状态:
- 按钮:背景变浅约 10%
- 任务卡:背景从 #2e2e2e 变为 #363636
- 链接:下划线出现

焦点状态:
- 蓝色轮廓(#4d9fff)用于键盘导航
- 2px 实心轮廓,4px 偏移
- 为键盘用户确保可访问性

活动/按下状态:
- 按钮:背景稍微变暗,缩小至 98%
- 创建触觉反馈

禁用状态:
- 降低不透明度(50%)
- 光标更改为不允许
- 文本颜色转移至 #666666

2. 图表/图形颜色:
问题:报告页面有图表(速度、燃尽图)。Jordan 显示了示例,但颜色可能需要调整。
关注:图表库(我们使用 Recharts)有默认颜色,在暗模式下不起作用。

需要决定:接受嵌入内容不会匹配主题?或者在暗模式下隐藏嵌入?

4. 移动应用程序:
问题:Jordan 为网络设计了这个。移动需要单独考虑。
问题:移动与暗模式一起推出?或发送移动而不初始暗模式?
Alex(移动 PM)需要权衡。

5. 营销网站:
问题:我们的营销网站(taskflow.com)应该有暗模式吗?
还是只是应用程序?
Jordan: "这是额外的范围。营销网站是单独的代码库。"

6. 默认模式:
问题:暗模式应该是选择加入还是选择退出?

技术实现说明:

Jordan 与 Jamie(工程主管)就实现方法进行了合作:

CSS 变量:
使用 CSS 自定义属性的颜色:
--background-primary: #1a1a1a;
--text-primary: #f0f0f0;
等等。

通过改变根 CSS 变量在浅/暗之间切换。

持久性:
用户首选项存储在数据库中(用户设置)
也存储在 localStorage 以获得即时加载

动画:
在切换时平滑过渡(0.2s 缓和)
Jordan: "浅色和暗色模式之间没有刺耳的闪现"

系统首选项检测:
使用 prefers-color-scheme 媒体查询
尊重 OS 级暗模式设置

每个组件覆盖:
某些组件可能需要特殊处理(图表、嵌入)
工程确定实施中的边缘情况
