# 用户访谈 #2：IC 工程师人物

**参与者：** David Chen
**职务：** 高级后端工程师
**公司：** GrowthLabs（180 名员工，融资 B 轮）
**SingTech 方案：** Pro（公司范围，120 个活跃用户）
**日期：** 2024 年 10 月 6 日
**访谈者：** 高级产品经理
**时长：** 28 分钟

---

## 背景

David 在 GrowthLabs 工作了 2 年。他从事后端基础设施工作，主要使用 Node.js 和 PostgreSQL。他的团队在 6 个月前从 Linear 切换到 SingTech。他是一个重度用户 - 每天都使用 SingTech。

---

## 访谈

**问：你在日常工作流程中如何使用 SingTech？**

David："早上第一件事就是检查 SingTech。看看分配给我的任务、今天截止的任务、优先级高的任务。我主要使用键盘快捷键 - 我几乎不用鼠标。Cmd+K 搜索，J 和 K 导航任务。速度很快，这对我很关键。如果工具速度慢，我就不会用它。"

**问：你最喜欢 SingTech 的什么？**

David："速度。真的，它加载速度非常快。没有加载指示器，没有延迟。我之前用过 Linear，它也很快，但 SingTech 感觉更快。GitHub 集成太棒了 - 我可以将 PR 链接到任务，合并时状态会自动更新。这对我来说非常重要。减少了手动更新。"

**问：你对 SingTech 有什么不满的地方？**

David："没有暗色主题！我有时工作到很晚，亮白色界面让我的眼睛很难受。我安装了浏览器扩展来强制使用暗色主题，但看起来很糟。这是我最想要的功能。另外，移动网页版本很不理想。我有时在手机上检查任务，体验不太好 - 需要大量滚动，难以阅读内容。"

**问：跟我说说你是如何获取任务背景的。**

David："背景信息非常重要。如果一个任务只是说'修复登录 bug'，没有详细信息，我会浪费时间搜索信息。我们团队中好的产品经理会写详细的描述 - 包括背景、边界情况、相关文档的链接。SingTech 在这方面表现最好。Markdown 支持很好 - 他们可以包括代码片段、链接，我需要的所有东西。"

**问：通知怎么样？**

David："哎呀。太多了。我收到每样东西的邮件。有人在任务上评论，我收到邮件。有人给我分配任务，我收到邮件。有人靠近我在看的任务，我收到邮件。我基本上关闭了大多数通知。我一天手动检查 SingTech 几次。应该有摘要模式 - 像一封包含所有更新的电子邮件，而不是 20 封单独的邮件。"

**问：团队协作怎么样？**

David："评论功能很好。我喜欢我可以 @提及人，他们会收到通知。线程功能很好，可以保持讨论有条理。但说实话，我们很多协作仍然在 Slack 中进行。我们在 Slack 中讨论，然后有人更新任务。如果 Slack 集成更好就太好了 - 像在 Slack 中发布，它自动为链接的任务添加评论。"

**问：如果你能添加一个功能，会是什么？**

David："暗色主题。毫无疑问。之后，也许是离线支持？我有时在飞机上工作，如果至少能离线查看任务就太好了。哦，还有 API 访问 - 我想自动化一些东西。像从错误监控警报自动创建任务。"

**问：SingTech 与 Linear 相比怎么样？**

David："Linear 对工程师来说 UX 稍微好一点 - 键盘快捷键更全面。而且他们有暗色主题！但 SingTech 对跨职能团队更好。我们的产品经理和设计师使用 SingTech，他们觉得 Linear 太以工程为中心。所以 SingTech 对整个团队效果更好。"

**问：还有其他反馈吗？**

David："搜索功能可以更好。目前它只是文本搜索。我想按自定义字段搜索，或者组合过滤器。像'显示分配给我的所有高优先级任务，在 API 项目中，本周截止日期'。我可以用过滤器实现，但搜索会更快。"

---

## 关键见解
- 速度/性能至关重要 - 继续使用的主要原因
- 暗色主题是 #1 功能请求（影响用户保留）
- GitHub 集成高度重视
- 键盘快捷键对重度用户来说必不可少
- 移动网页体验不足
- 通知过载 - 需要摘要模式
- 任务背景（详细描述）非常重要
- Slack 集成可以更好（双向同步）
- 搜索功能可以更强大
- 感赏 Markdown 支持以提供技术细节
