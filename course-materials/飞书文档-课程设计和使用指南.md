## 🏗️ 课程设计哲学

### 1️⃣ 配置驱动架构

**核心思想**：一个 JSON 文件控制所有内容

```
course-structure.json
  ↓
  控制模块顺序、命令路由、网站导航
```

**为什么这样设计？**

✅ **灵活性** - 添加新模块只需改配置  
✅ **可维护** - 单一真理来源  
✅ **可扩展** - 重新排序无需改代码

**实际效果**：
- 10 个斜杠命令都是**同一个文件**
- 添加模块 = 编辑 JSON + 创建文件夹
- 重新排序 = 改 JSON，其他自动更新

---

### 2️⃣ 斜杠命令系统

**所有命令都相同**，通过解析命令名找到对应模块

```
用户输入: /start-1-2
  ↓
解析模块ID: "1.2"
  ↓
读取配置: course-structure.json
  ↓
加载脚本: lesson-modules/1.2-visualizing-files/CLAUDE.md
  ↓
执行教学
```

**好处**：
- 无需维护 10 个不同的命令文件
- 添加新模块只需在配置中注册
- 命令逻辑统一，bug 少

---

### 3️⃣ 教学脚本结构

每个模块的 `CLAUDE.md` 都是**逐字教学脚本**

**关键元素**：
- **Say:** - Claude 要说的原话（逐字输出）
- **Action:** - Claude 要执行的操作
- **Stop:** - 停止等待学生回应
- **Check:** - 检查学生是否完成

**示例**：
```markdown
**Say:**
"欢迎来到模块 1.2！今天我们要学习..."

**Action:**
列出当前目录的所有文件

**Stop: 等待学生确认**

**Check:** 学生说"准备好了"
```

**为什么逐字？**
- 确保教学一致性
- 学生体验统一
- 易于维护和更新

---

### 4️⃣ 动态变量系统

**不使用硬编码**，使用变量自动计算

**常用变量**：
- `{moduleId}` - 当前模块 ID
- `{moduleTitle}` - 模块标题
- `{nextModuleId}` - 下一个模块 ID
- `{nextCommand}` - 下一个命令

**条件逻辑**：
- `{ifLastInLevel}...{/ifLastInLevel}` - 级别最后一个模块
- `{ifNotLastInCourse}...{/ifNotLastInCourse}` - 课程未结束

**好处**：
- 重新排序模块，脚本自动更新
- 添加/删除模块无需修改现有脚本
- 一次编写，处处适用

## 💡 核心概念速记

### 🤖 代理 (Agents)

**定义**：同时工作的独立 Claude 实例

**效果**：
```
传统方式: 10 个任务 × 5 分钟 = 50 分钟
代理方式: 10 个任务同时处理 = 5 分钟
```

**实际应用**：
- 批量处理会议记录
- 并行研究竞争对手
- 分析多个数据源

---

### 👥 子代理 (Sub-agents)

**定义**：预配置的专业角色

**示例角色**：
- 👨‍💻 **工程师** - 技术可行性审查
- 💼 **高管** - 战略对齐审查
- 👤 **用户研究员** - 用户需求验证

**实际应用**：
- PRD 多角度审查
- 功能规格质量检查
- 获得专业反馈

---

### 🧠 项目记忆 (CLAUDE.md)

**定义**：项目级的上下文和规则

**示例**：
```markdown
# SingTech 项目上下文

## 我们的风格
- 使用简洁的语言
- PRD 不超过 2 页
- 总是包含用户故事

## 我们的工具
- Jira 用于需求跟踪
- Figma 用于设计
```

**实际应用**：
- 保持团队风格一致
- 避免重复说明
- 提高协作效率

---

### @-Mentions

**定义**：引用文件提供完整上下文

**用法**：`@文件名.md`

**效果**：Claude 看到完整文件内容，无需手动复制粘贴

## 🎯 实际工作场景应用

### 场景 1: 处理每周会议记录

**问题**：10 个会议的粗略记录，需要提取行动项

**解决方案**：
```bash
claude
"我有 10 个会议记录文件，请并行处理它们，
提取每个会议的行动项、决策和后续步骤"
```

**学习自**：Module 1.4 (代理)

---

### 场景 2: 写新功能的 PRD

**问题**：需要为新功能写 PRD，但不确定从何开始

**解决方案**：
```bash
# 复制模板
cp lesson-modules/2.1-write-prd/prd-templates/Carls-PRD-Template.md my-prd.md

claude
"帮我用苏格拉底提问法深化对这个功能的思考，
然后填写 PRD 模板"
```

**学习自**：Module 2.1 (撰写 PRD)

---

### 场景 3: 竞争对手研究

**问题**：需要快速了解 5 个竞争对手

**解决方案**：
```bash
claude
"启动 5 个代理并行研究以下竞争对手：
Asana, Linear, Monday, ClickUp, Jira

每个代理收集：功能特点、定价、目标用户、优势劣势
然后创建竞争格局综合报告"
```

**学习自**：Module 1.4 (代理)

---

### 场景 4: 多角度审查文档

**问题**：写好了 PRD，需要从不同角度审查

**解决方案**：
```bash
claude
"创建三个专业审查角色：
1. 工程师 - 评估技术可行性
2. 高管 - 评估战略对齐
3. UX 设计师 - 评估用户体验

用这三个角色审查我的 PRD"
```

**学习自**：Module 1.5 (自定义子代理)

## 🔧 故障排除

### 问题 1: 斜杠命令不工作

```bash

ls -la .claude/

# 3. 重启 Claude
exit
claude
```
