# Claude Code 产品经理课程 - 完整使用指南

## 📚 课程概述

这是一个**互动式、实践型**的课程，教产品经理如何使用 Claude Code 来提升日常工作效率。

### 课程特点

- ✅ **由 Claude 直接教学** - 不是看视频或文档，而是与 AI 导师互动学习
- ✅ **边做边学** - 通过真实的 PM 工作场景练习
- ✅ **完整的虚拟公司** - 使用 SingTech 这个虚拟K歌娱乐公司作为实践背景
- ✅ **配置驱动架构** - 所有模块通过斜杠命令启动，无需记忆复杂命令

### 💡 核心概念速记

**代理 (Agents)**
- **定义**: 同时工作的独立 Claude 实例
- **用途**: 并行处理多个相似任务
- **效果**: 10 个任务 × 5 分钟 = 5 分钟（而非 50 分钟）

**子代理 (Sub-agents)**
- **定义**: 预配置的专业角色
- **用途**: 多角度审查和反馈
- **示例**: 工程师、高管、用户研究员

**项目记忆 (CLAUDE.md)**
- **定义**: 项目级的上下文和规则
- **用途**: 保持团队风格一致
- **位置**: 项目根目录的 CLAUDE.md

**@-Mentions**
- **定义**: 引用文件提供完整上下文
- **用法**: `@文件名.md`
- **效果**: Claude 看到完整文件内容

## 🛠️ 课程架构详解

### 配置驱动系统

课程使用**单一配置文件**控制所有内容：

**核心文件**: `course-structure.json`

```json
{
  "version": "1.0.0",
  "levels": [
    {
      "id": "1",
      "name": "基础",
      "modules": [
        {
          "id": "1.1",
          "title": "欢迎",
          "path": "lesson-modules/1.1-welcome/CLAUDE.md",
          "command": "start-1-1"
        }
      ]
    }
  ]
}
```

### 斜杠命令系统

所有 10 个斜杠命令都是**完全相同**的：

**命令位置**: `.claude/commands/start-*.md`

**工作原理**:
1. 用户输入 `/start-1-2`
2. 命令解析出模块 ID: "1.2"
3. 读取 `course-structure.json` 找到对应路径
4. 加载 `lesson-modules/1.2-visualizing-files/CLAUDE.md`
5. 执行教学脚本

**好处**:
- 添加新模块只需编辑配置文件
- 重新排序模块无需改动命令
- 维护简单，扩展容易

### 教学脚本结构

每个 CLAUDE.md 都是一个**逐字教学脚本**：

**关键元素**:
- **Say:** - Claude 要说的话（逐字输出）
- **Action:** - Claude 要执行的操作
- **Stop:** - 停止并等待学生回应
- **Check:** - 检查学生是否完成

**示例**:
```markdown
**Say:**
"欢迎来到模块 1.2！今天我们要学习..."

**Action:**
列出当前目录的所有文件

**Stop: 等待学生确认**

**Check:** 学生说"准备好了"
```

### 动态变量系统

教学脚本使用**变量**而非硬编码：

**常用变量**:
- `{moduleId}` - 当前模块 ID (如 "1.2")
- `{moduleTitle}` - 模块标题 (如 "可视化文件")
- `{nextModuleId}` - 下一个模块 ID
- `{nextCommand}` - 下一个命令 (如 "/start-1-3")
- `{levelId}` - 当前级别 (如 "1")

**条件逻辑**:
- `{ifLastInLevel}...{/ifLastInLevel}` - 如果是该级别最后一个模块
- `{ifNotLastInCourse}...{/ifNotLastInCourse}` - 如果不是课程最后一个

**好处**:
- 重新排序模块，脚本自动更新
- 添加/删除模块无需修改现有脚本

## 💼 实际工作场景应用

### 场景 1: 处理每周会议记录

**问题**: 你有 10 个会议的粗略记录，需要提取行动项

**解决方案**:
```bash
cd your-project
claude

# 告诉 Claude:
"我有 10 个会议记录文件，请并行处理它们，
提取每个会议的行动项、决策和后续步骤，
在每个文件末尾添加摘要"
```

**学习自**: Module 1.4 (代理)

---

### 场景 2: 写新功能的 PRD

**问题**: 需要为新功能写 PRD，但不确定从何开始

**解决方案**:
```bash
# 复用课程模板
cp course-materials/lesson-modules/2.1-write-prd/prd-templates/Carls-PRD-Template.md my-feature-prd.md

claude

# 告诉 Claude:
"帮我用苏格拉底提问法深化对这个功能的思考，
然后填写 PRD 模板"
```

**学习自**: Module 2.1 (撰写 PRD)

---

### 场景 3: 竞争对手研究

**问题**: 需要快速了解 5 个竞争对手

**解决方案**:
```bash
claude

# 告诉 Claude:
"启动 5 个代理并行研究以下K歌行业竞争对手：
全民K歌（腾讯）、唱吧、视易、酷狗音乐、天籁K歌

每个代理应该通过网络搜索收集：
- 产品特点和功能
- 市场定位和目标用户
- 竞争优势和劣势
- 最新动态和战略方向

然后创建一个竞争格局综合报告"
```

**学习自**: Module 1.4 (代理)

---

### 场景 4: 多角度审查文档

**问题**: 写好了 PRD，需要从不同角度审查

用这三个角色审查我的 PRD"

## 🔧 故障排除

### 问题 1: 斜杠命令不工作

**症状**: 输入 `/start-1-1` 没反应

**解决**:
```bash
# 1. 确认在正确目录
pwd  # 应显示 .../course-materials

# 2. 确认 .claude 文件夹存在
ls -la .claude/

# 3. 确认命令文件存在
ls -la .claude/commands/

# 4. 重启 Claude
exit
claude
```

---

### 问题 2: Claude 说看不到文件

**症状**: Claude 找不到 company-context/ 文件

**解决**:
```bash
# 确认文件确实存在
ls company-context/

# 使用绝对路径
cat /完整路径/course-materials/company-context/COMPANY.md

# 或使用 @-mention
# 在 Claude Code 中输入 @company-context/COMPANY.md
```

---

### 问题 3: 翻译不完整

**症状**: 某些文件还是英文

**解决**:
```bash
# 运行验证脚本
./verify-translation.sh

# 查看具体文件
head -30 有问题的文件路径.md

# 如果需要，重新翻译该文件
```

---

### 问题 4: 模块跳转不工作

**症状**: 完成模块后不知道下一步

**解决**:
每个模块结束时，Claude 会告诉你下一个命令。也可以查看课程结构：

```bash
cat course-structure.json

# 或直接按顺序输入
/start-1-1
/start-1-2
/start-1-3
# 等等
```
